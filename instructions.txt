render(<MyAwesomeApp />);

# Para ver el DOM de forma estructurada
screen.debug();

# Estructura de un test suite y test case
describe('add', () => {
    test('should add two positives numbers', () => {
        // ! 1. Arrange
        const a = 1;
        const b = 2;

        // ! 2. Act
        const result = add(a, b);

        // ! 3. Assert
        expect(result).toBe(3);
    });
});

****NOTAS****
Dentro de las llaves {} se colocan expresiones de JavaScript

Los booleanos no se imprimen en React

Si los valores como const no cambian dentro del componente, es major dejarlo afuera

JSX usa camelCase

'rafc' para crear rapidamente un componente

Usar ? para hacer opcional la property

Comentar codigo: Ctrl + K > Ctrl + C

Los eventos empieza con "on"

***USE STATE***
const [count, setCount] = useState(quantity);

Debe tener un evento que dispara la funcion setCount (para evitar infinitos renders) 

const handleClick = () => {
    setCount(count + 1);
}

setCount es la function que Vuelve a renderizar el componente
